## 2024-05-24 - [Path Traversal in Plugin Manager]
**Vulnerability:** The plugin manager endpoints (`installPlugin`, `updatePlugin`, `uninstallPlugin`) took a `name` from a route parameter or derived it from a repository's `package.json` and used it directly in filesystem paths (e.g., `path.join(PLUGINS_DIR, name)`) without structural validation. Additionally, space checks on `URL.pathname` failed due to automatic `%20` URL-encoding by Node's `new URL()`.
**Learning:** URL parsing (`new URL`) encodes characters like spaces into `%20`, bypassing simple `.includes(" ")` checks. Concatenating user-controlled names to paths allows Arbitrary File/Directory Deletion and Arbitrary Code Execution (via `pnpm install` in traversed directories).
**Prevention:** Always use an allowlist regex for filenames and identifiers derived from user input or external files (`/^(llm|message|memory|tool)-[a-zA-Z0-9_-]+$/`). For URL-validation against specific characters, account for URL encoding.
