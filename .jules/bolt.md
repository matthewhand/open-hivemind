## 2025-01-28 - ToolRegistryPanel Optimization Rejection
**Learning:** Moving an O(N) array search `find()` from an `onClick` event handler (a cold path) into a `useMemo` map generation on the component body (the hot path render cycle) degrades performance rather than improving it. Click handlers execute negligibly fast, whereas allocating memory for a new Map on every state change during render is an anti-pattern.
**Action:** Always ensure that `useMemo` pre-computed maps are only used to replace `find()` or `includes()` calls that exist *inside* `.map()` loops directly in the render cycle, never for cold paths like user event handlers.
