## 2026-03-09 - [O(1) lookups over O(N) mapping in get methods]
**Learning:** `getBot(id)` in `BotManager` was triggering an expensive O(N) map of *all* configured bots on every single request because it relied on `getAllBots()`. This meant full environment override resolution and config sanitization for every bot in the system, just to return one.
**Action:** When working on retrieval methods, directly target the underlying storage/map mechanisms (like `botConfigManager.getBot(id)`) and selectively apply transformation logic to the single result, skipping full-collection getters.

## 2024-03-16 - Prevent Redundant User Fetching in message-mattermost
**Learning:** Found a missing performance parity in `packages/message-mattermost/src/MattermostService.ts` for fetching profile data. Although it was mitigated in `src/integrations/mattermost/MattermostService.ts`, the external decoupled package suffered from the same bottleneck by consecutively polling Mattermost profile endpoints for multiple messages belonging to an identical user without caching.
**Action:** Implemented an in-memory `userCache` Dictionary structured using `USER_CACHE_TTL` inside the class. Refactored the array iterator to check the dictionary before scheduling identical tasks, decreasing external execution times dramatically in instances of batch pulling chat logs.

## 2024-03-16 - Cache Stampede in Mattermost Promise Coalescing
**Learning:** Initially implemented a naive user profile cache using the awaited resolved value in a `Promise.all` `.map` closure. This is a subtle flaw: because all async requests execute synchronously *before* awaiting sequentially within the event loop, any batch of duplicated missing users evaluates the cache lookup before any single entry concludes fetching, resulting in a cache stampede (multiple duplicate HTTP fetches per unique ID).
**Action:** Replaced resolving caches with Promise caching directly into the Memory Dictionary. `userCache` now immediately stores the Promise footprint of `client.getUser()` synchronously. When identical subsequent users request the lookup in that instant `.map()` resolution batch, they instantly await on the *identical initial Promise execution*, ensuring single-requests network execution while avoiding race conditions. Included a maximum bounds check on the dictionary to prevent unbounded cache inflation.
