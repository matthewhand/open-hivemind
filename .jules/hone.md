## 2026-07-14 | [Architectural Audit] | Insight: Div elements heavily used for interactive components and lists (div-soup), causing accessibility and keyboard navigation regressions. Card actions wrapping elements inappropriately caught click propagation. Missing empty states resulted in blank UI sections when no data was available. | Protocol: Consistently enforce semantic HTML elements (buttons) for interactive components, explicitly handle event propagation directly on interactive children, and wrap logic in structured \<EmptyState\> blocks for graceful fallback.
