## 2024-06-22 | [Architectural Audit] | Insight: Empty states were built using generic <div> elements with missing semantic meaning for screen readers. | Protocol: Refactored `EmptyState` component to use `<section>` with `aria-labelledby` referencing an accessible heading (`<h2>`) to ensure correct document outline and discoverability.

## 2024-06-22 | [Architectural Audit] | Insight: The "Div-soup" pattern was prevalent in list and table actions across multiple pages (`DataTable`, `MemoryProvidersPage`, `MessageProvidersPage`, `ResponseProfilesPage`, `GuardsPage`, `ToolProvidersPage`, `LLMProvidersPage`), trapping click events on non-semantic `<div>` containers using `onClick={(e) => e.stopPropagation()}` to prevent row expansion. This violates keyboard accessibility and HTML semantics. | Protocol: Removed `onClick` from the `<div>` wrappers and applied `e.stopPropagation()` strictly within the `onClick` handler of the interactive `<Button>` or `<Toggle>` children, allowing them to appropriately handle both keyboard (`Enter`/`Space`) and mouse interactions.

## 2024-06-22 | [Architectural Audit] | Insight: Async test connection loading states in lists (e.g. `MemoryProvidersPage`) relied on overloading complex data types (e.g. `TestResult | 'loading'`) to represent transient states, increasing the risk of race conditions and state corruption. | Protocol: Implemented deterministic state-machine maps (e.g., `testingStates: Record<string, boolean>`) to accurately isolate and manage button loading states per-item without polluting the payload state.
