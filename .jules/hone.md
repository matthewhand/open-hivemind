## 2025-06-27 | [Architectural Audit] | Insight: Several core UI components use non-semantic div elements with role="button" instead of native button elements, compromising accessibility and keyboard operability. | Protocol: Replaced all generic div structural wrappers acting as triggers (HamburgerMenu, ThemeDropdown, AdvancedThemeSwitcher, Dropdown, NavbarWithSearch, Accordion) with native `<button type="button">` to inherently handle Enter/Space keys, focus states, and disabled attributes naturally.

## 2025-06-27 | [Architectural Audit] | Insight: Modals and overlay drawers manipulate document.body.style.overflow to prevent scrolling, but arbitrarily reset it to an empty string on unmount. This breaks if multiple overlays overlap or if an existing overflow state existed prior to mounting. | Protocol: Implemented defensive state-retention in DetailDrawer by capturing the previous overflow state prior to locking, and strictly restoring that exact previous state upon cleanup.

## 2025-06-27 | [Architectural Audit] | Insight: Auxiliary actions and secondary information are hidden behind hover interactions (e.g. opacity-0 group-hover:opacity-40). This causes elements to remain hidden for keyboard-only or screen-reader users attempting to navigate to them. | Protocol: Augmented brittle hover classes across interactive components (AdvancedThemeSwitcher, NavbarWithSearch) by injecting `focus-visible:opacity-100`, `group-focus-within:opacity-40`, and `group-focus-visible:opacity-40` to ensure keyboard focus visibility.
