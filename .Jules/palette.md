## 2025-03-07 - Add ARIA Labels to Icon-Only Buttons
**Learning:** This application heavily utilizes custom DaisyUI wrapper components, often creating `<button>` elements that only contain HeroIcons or Lucide React icons. By default, these icon-only buttons are invisible to screen readers, causing accessibility issues where users cannot determine the button's purpose (e.g., delete, edit, close, expand).
**Action:** When implementing new UI elements or refactoring existing ones, always ensure that any `<button>` lacking visible, descriptive text content includes an `aria-label` attribute describing its function. When a button toggles state (like expanding a section), use `aria-expanded` in conjunction with a dynamic `aria-label`.
## 2026-03-10 - Prevent Focus Loss on Inline Buttons
**Learning:** When building custom input components that include inline action buttons (like clear, undo, or remove tags) inside the input container, clicking those buttons causes the active input element to lose focus (blur). This creates a poor user experience as it interrupts typing and often prematurely triggers `onBlur` commit events.
**Action:** Always add `onMouseDown={(e) => e.preventDefault()}` to inline action buttons within custom inputs. This intercepts the mouse event before it triggers a focus shift, allowing the subsequent `onClick` handler to fire while keeping the user's cursor perfectly retained in the text field.
## 2026-03-14 - Prevent CI Build Failures with pnpm Exec
**Learning:** In a strict `pnpm` workspace, using `npm run` or `npx` in shell scripts (like `build-netlify.sh` or `build-frontend.sh`) or inside `package.json` scripts (like `tsc && vite build`) can cause catastrophic build failures during CI/CD. The CI runners may not correctly resolve the global or local binary paths (e.g., `tsc` or `vite`) when executed via an alien package manager.
**Action:** Always replace `npm run` with `pnpm run` and `npx <tool>` with `pnpm exec <tool>` in all build scripts and `package.json` configurations to ensure deterministic module resolution within the pnpm workspace.
